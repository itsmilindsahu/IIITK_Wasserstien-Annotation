What This Project Does

The standard way to combine multiple annotators' video importance scores is to just average them Î“Ă‡Ă¶ but that arithmetic mean lives in unconstrained Euclidean space and ignores the fact that importance scores are probability distributions (they must sum to 1 and stay non-negative). When annotators strongly disagree, the mean can produce a bimodal "ghost" distribution that no annotator actually intended.

This project implements three geometric fixes on top of the Shang et al. (AAAI-25) video summarisation diffusion model, so every operation stays on the probability simplex â•¬Ă¶:

Fix 1
Wasserstein Barycenter
Replace arithmetic mean with an entropic Sinkhorn barycenter Î“Ă‡Ă¶ the true geometric center of distributions.
Fix 2
Dirichlet Forward Process
Replace Gaussian noise with Dirichlet noise + simplex projection so noisy states never leave the simplex.
Fix 3
Wasserstein Loss
Add a WÎ“Ă©Ă© distance penalty to the training loss so the model is penalised for distributional mistakes, not just pixel-wise MSE.

Utility code/sinkhorn.py
plain english

This is the simplex projector Î“Ă‡Ă¶ a small utility that takes any vector (which might have negative numbers or not sum to 1) and forces it back onto the probability simplex, i.e. makes it a valid probability distribution.

It does this in two cheap steps repeated a handful of times: first it clips every negative value to a tiny positive number (you can't have negative probability), then it divides by the total sum so everything adds up to exactly 1. That's it Î“Ă‡Ă¶ no optimisation, no matrix inversions.

The comment in the file is honest about naming: the proposal calls this step "Sinkhorn projection" (Proj_â•¬Ă¶), but the actual Sinkhorn algorithm Î“Ă‡Ă¶ the fancy iterative OT solver Î“Ă‡Ă¶ lives in barycenter.py. This file is just clip+renorm.

sinkhorn.py Proj_â•¬Ă¶: clip negatives Î“ĂĄĂ† renorm to sum=1 
import numpy as np



# Quick simplex projector. Called "sinkhorn" because the proposal calls

# the Proj_delta step that name Î“Ă‡Ă¶ the actual Sinkhorn OT solver lives in

# barycenter.py. This is just clip + renorm.

def sinkhorn_projection(x, n_iter=5):

    x = np.array(x, dtype=float)

    for i in range(n_iter):

        x = np.clip(x, 1e-8, None)   # remove negatives

        x = x / x.sum()                # renorm to probability

    return x





def test():

    v = np.array([0.5, -0.2, 0.9, 0.0])

    out = sinkhorn_projection(v)

    print(out, out.sum())  # Î“ĂĄĂ† [0.29 0.0 0.52 0.0] 1.0 roughly



if __name__ == "__main__":

    test()
line by line

clip(x, 1e-8, None) Î“Ă‡Ă¶ sets any value below 1e-8 to 1e-8, removing negatives while keeping everything strictly positive (needed for log() calls downstream).

x / x.sum() Î“Ă‡Ă¶ divides every element by the total so the vector sums to exactly 1. Running this in a short loop ensures numerical stability (a single pass is fine in practice).

n_iter=5 Î“Ă‡Ă¶ repeated 5 times by default. After clipping and renorming once the vector is already on the simplex; extra iterations are just a cheap stability guard.

Fix 1 code/barycenter.py
plain english

This file answers one question: "what is the single best distribution that represents all the annotators at once?"

The naive answer is the arithmetic mean Î“Ă‡Ă¶ just add them up and divide. But the arithmetic mean lives in Euclidean space and ignores the shape of the distributions. If half the annotators peak around frame 5 and half peak around frame 22, the arithmetic mean peaks at both Î“Ă‡Ă¶ a bimodal blob that no annotator intended.

The Wasserstein barycenter is the correct answer: the distribution whose average transport cost to all annotators is minimised. Intuitively it "slides" mass to an interpolated mode, rather than smearing it across two modes.

The algorithm here is the entropic Sinkhorn barycenter Î“Ă‡Ă¶ an iterative algorithm that alternately updates scaling vectors u, v and renorms the barycenter until convergence. It uses only numpy (no POT library), kept to ~15 lines on purpose.

barycenter.py Fix 1 Î“Ă‡Ă¶ entropic Sinkhorn barycenter + arithmetic mean baseline 
import numpy as np

from sinkhorn import sinkhorn_projection



# Real entropic Sinkhorn barycenter (not the simplified projector above)



def cost_matrix(N):

    idx = np.arange(N)

    C = (idx[:, None] - idx[None, :]) ** 2   # squared frame-distance

    return C / C.max()                         # normalise to [0, 1]





def wasserstein_barycenter(annotator_scores, n_iter=40, reg=0.05):

    """K â”śĂą N annotator score matrix Î“ĂĄĂ† 1 â”śĂą N barycenter on the simplex."""

    K, N = annotator_scores.shape

    C    = cost_matrix(N)

    Kmat = np.exp(-C / reg)            # Gibbs kernel  K = exp(-C/â•¬â•ˇ)



    u    = np.ones((K, N))

    bary = np.ones(N) / N              # uniform start



    for it in range(n_iter):

        v    = annotator_scores / (Kmat.T @ u.T).T.clip(1e-16)

        Ku   = (Kmat @ v.T).T

        bary = np.exp(np.mean(np.log(Ku * u + 1e-16), axis=0))

        bary = sinkhorn_projection(bary, n_iter=1)

        u    = bary[None, :] / Ku.clip(1e-16)



    return bary





def arithmetic_mean(annotator_scores):

    return annotator_scores.mean(axis=0)   # the naâ”śÂ»ve baseline
line by line

cost_matrix(N) Î“Ă‡Ă¶ builds an Nâ”śĂąN matrix where entry (i,j) = (iÎ“ĂŞĂ†j)â”¬â–“ / max. This is the "ground cost" of moving probability mass from frame i to frame j. Squaring means big jumps are penalised disproportionately.

Kmat = exp(Î“ĂŞĂ†C/reg) Î“Ă‡Ă¶ the Gibbs kernel. The regularisation parameter reg=0.05 controls how blurry the optimal transport plan is. Smaller reg = sharper (harder to compute), larger reg = more blurry but faster convergence.

The loop (40 iterations) Î“Ă‡Ă¶ alternates two Sinkhorn scaling steps. v matches each annotator's marginal; Ku is the transported version; bary is updated as the geometric mean of the scaled kernels, then projected back to the simplex. This is the standard free-support Sinkhorn barycenter algorithm.

arithmetic_mean Î“Ă‡Ă¶ kept for direct comparison. One line, dead simple, and geometrically wrong for distributions with different support.

Fix 2 code/forward.py
plain english

Diffusion models work by progressively adding noise to the data, then training a network to reverse the process. The original model uses Gaussian noise. The problem: Gaussian noise can push values negative or make them sum to something other than 1, which means the noisy state is no longer a valid distribution Î“Ă‡Ă¶ it has left the simplex.

This file replaces Gaussian noise with Dirichlet noise. A Dirichlet sample is always non-negative and already sums to 1, so it lives on the simplex by construction. After mixing the signal x_star and the Dirichlet noise according to the cosine diffusion schedule, we apply the simplex projector from sinkhorn.py (the Proj_â•¬Ă¶ step) as an extra safety net to correct any floating-point drift.

The baseline forward step is kept alongside for comparison Î“Ă‡Ă¶ it uses Gaussian noise and only does a crude clip+renorm, not a proper simplex projection.

forward.py Fix 2 Î“Ă‡Ă¶ Dirichlet forward process + Gaussian baseline 
import numpy as np

from sinkhorn import sinkhorn_projection





def alpha_bar(t, T=200, s=0.008):

    # Standard cosine noise schedule Î“Ă‡Ă¶ same trick as image diffusion models.

    # alpha_bar(0) Î“Ă«ĂŞ 1 (pure signal), alpha_bar(T) Î“Ă«ĂŞ 0 (pure noise).

    f = lambda tt: np.cos(((tt / T) + s) / (1 + s) * np.pi / 2) ** 2

    return f(t) / f(0)





def dirichlet_noise(N, alpha=1.0):

    # Uniform Dirichlet sample Î“Ă‡Ă¶ always on the simplex.

    return np.random.dirichlet(alpha * np.ones(N))





def forward_step_mine(x_star, t, T=200):

    ab    = alpha_bar(t, T)

    delta = dirichlet_noise(len(x_star))

    # Mix signal and noise according to the diffusion schedule

    xt    = np.sqrt(ab) * x_star + np.sqrt(1 - ab) * delta

    # Proj_â•¬Ă¶ step Î“Ă‡Ă¶ brings xt back to the simplex (Equation 4 in proposal)

    xt    = sinkhorn_projection(xt, n_iter=3)

    return xt





def forward_step_baseline(x_star, t, T=200):

    # Original AAAI-25 style Gaussian forward Î“Ă‡Ă¶ kept for comparison

    ab    = alpha_bar(t, T)

    noise = np.random.randn(len(x_star)) * 0.1

    xt    = np.sqrt(ab) * x_star + np.sqrt(1 - ab) * noise

    xt    = np.clip(xt, 0, None)           # crude fix: remove negatives

    s     = xt.sum()

    xt    = xt / s if s > 0 else np.ones_like(xt) / len(xt)

    return xt
line by line

alpha_bar(t, T) Î“Ă‡Ă¶ cosine noise schedule. At t=0 you get almost pure signal (â•¬â–’â• Ă Î“Ă«ĂŞ1); at t=T you get almost pure noise (â•¬â–’â• Ă Î“Ă«ĂŞ0). The cosine shape gives a smoother transition than a linear schedule and is standard in modern diffusion models (DDPM++, etc.).

dirichlet_noise(N, alpha=1.0) Î“Ă‡Ă¶ draws a sample from a symmetric Dirichlet distribution with concentration 1.0. With alpha=1 this is the uniform distribution over the simplex Î“Ă‡Ă¶ every point on â•¬Ă¶ is equally likely. This is the natural "maximum entropy" noise for simplex-valued data.

np.sqrt(ab) * x_star + np.sqrt(1Î“ĂŞĂ†ab) * delta Î“Ă‡Ă¶ the diffusion interpolation. As t grows, alpha_bar shrinks so the noise term dominates. The square-root ensures the variance is correct under the Gaussian-analogue derivation.

forward_step_baseline Î“Ă‡Ă¶ the thing we are fixing. Notice the clip(0, None) plus manual renorm Î“Ă‡Ă¶ it gets the state back onto the simplex but has no theoretical justification for why that particular projection is the right one. Our Dirichlet version is principled.

Fix 3 code/losses.py
plain english

The standard training loss is Mean Squared Error (MSE) Î“Ă‡Ă¶ it penalises the model if its predicted noise vector differs from the target noise vector element-by-element. But MSE treats all dimensions as independent and doesn't care about the shape of the distribution. A prediction that shifts all probability mass two frames to the left gets the same MSE penalty whether those frames are far apart or close together.

The Wasserstein loss adds a WÎ“Ă©Ă© distance term that does care about the shape: it measures how far you'd have to move mass to transform the predicted distribution into each annotator's distribution. The final loss is MSE + â•¬â•— â”śĂą WÎ“Ă©Ă©, where â•¬â•—=0.1 is a small regularisation weight.

The WÎ“Ă©Ă© distance is approximated here using a closed-form 1D formula: the sum of squared differences of CDFs. Note from the README: this proxy may not be equivalent to the true WÎ“Ă©Ă© Î“Ă‡Ă¶ see the open question below. But for Phase 1 it's fast and torch-free.

losses.py Fix 3 Î“Ă‡Ă¶ Wasserstein-regularised loss + plain MSE baseline 
import numpy as np





def w2_1d(p, q):

    # Closed-form W2 between two 1D distributions via CDFs.

    # True 1D W2 = integral of squared quantile differences.

    # This proxy uses squared CDF differences Î“Ă‡Ă¶ see open question below.

    P = np.cumsum(p)

    Q = np.cumsum(q)

    return np.sum((P - Q) ** 2)





def wasserstein_loss(eps, eps_hat, x0_hat, annotator_scores, lam=0.1):

    mse      = np.mean((eps - eps_hat) ** 2)

    w2_terms = [w2_1d(x0_hat, a) for a in annotator_scores]

    w2       = np.mean(w2_terms)        # average over all annotators

    total    = mse + lam * w2

    return total, mse, w2
line by line

np.cumsum(p) Î“Ă‡Ă¶ computes the Cumulative Distribution Function (CDF) of distribution p. The CDF at index k is the total probability mass in frames 0..k.

sum((PÎ“ĂŞĂ†Q)â”¬â–“) Î“Ă‡Ă¶ the squared L2 distance between the two CDFs. For 1D distributions, the true Wasserstein-2 distance is Î“ĂŞÂ˝|FÎ“ĂĽâ•—â”¬â•Ł(u)Î“ĂŞĂ†GÎ“ĂĽâ•—â”¬â•Ł(u)|â”¬â–“ du, which equals Î“ĂŞÂ˝|F(x)Î“ĂŞĂ†G(x)|â”¬â–“ dx only under specific conditions. The open question is whether the CDF formula used here matches what the Sinkhorn barycenter minimises.

lam=0.1 Î“Ă‡Ă¶ the regularisation coefficient â•¬â•—. Small enough that the MSE term still dominates training but the WÎ“Ă©Ă© penalty adds distributional shape awareness.

wasserstein_loss returns (total, mse, w2) Î“Ă‡Ă¶ the three components are logged separately so we can check the gradient signal. The raw "mine" loss reported in charts includes the â•¬â•—â”¬â•–WÎ“Ă©Ă© term, so it's numerically larger than the baseline MSE Î“Ă‡Ă¶ that's expected, not a bug.

Model code/model.py
plain english

The proposal calls for a Transformer as the noise-prediction network. But for Phase 1 Î“Ă‡Ă¶ a proof-of-concept that just needs to confirm the pipeline runs end-to-end Î“Ă‡Ă¶ a transformer is overkill. Instead, this file implements a single linear layer: one weight matrix and one bias vector, updated with hand-written numpy gradient descent.

The inputs are the noisy state xt (a length-N vector) and the timestep t (a scalar). The network multiplies xt by the weight matrix, adds a bias scaled by the normalised timestep, and returns a predicted noise vector. The gradient update is just plain SGD Î“Ă‡Ă¶ no momentum, no Adam.

This is explicitly a stand-in. Swapping it out for the Transformer in Phase 2 is not a config flag Î“Ă‡Ă¶ it requires proper architecture work.

model.py Linear noise predictor Î“Ă‡Ă¶ stand-in for the Phase 2 Transformer 
import numpy as np



# Stand-in for the Transformer noise predictor from the proposal.

# Using a linear model: this is just Phase 1. Swap in Phase 2.

class NoisePredictor:

    def __init__(self, N, lr=0.05):

        self.N  = N

        self.W  = np.random.randn(N, N) * 0.01   # weight matrix Nâ”śĂąN

        self.b  = np.zeros(N)                     # bias vector length N

        self.lr = lr                               # learning rate



    def forward(self, xt, t):

        tt = t / 200.0                            # normalise timestep to [0,1]

        return self.W @ xt + self.b * tt         # linear prediction



    def step(self, xt, t, target):

        pred    = self.forward(xt, t)

        gradOut = 2 * (pred - target) / self.N    # MSE gradient at output

        gW      = np.outer(gradOut, xt)            # gradient w.r.t W

        gb      = gradOut * (t / 200.0)           # gradient w.r.t b

        self.W -= self.lr * gW                    # SGD update

        self.b -= self.lr * gb

        return pred
line by line

W = randn(N,N) * 0.01 Î“Ă‡Ă¶ small random initialisation. Multiplying by 0.01 keeps early predictions near zero, preventing exploding gradients at the start of training.

forward: W @ xt + b * tt Î“Ă‡Ă¶ a linear projection of the noisy state xt, with bias modulated by the normalised timestep tt. The timestep conditioning is crude (just a scalar multiply on the bias) Î“Ă‡Ă¶ a real model would use sinusoidal position embeddings or learned timestep embeddings.

step: gradOut = 2*(predÎ“ĂŞĂ†target)/N Î“Ă‡Ă¶ the MSE gradient at the output. The factor 2 comes from differentiating (predÎ“ĂŞĂ†target)â”¬â–“, and dividing by N normalises by the number of frames.

gW = outer(gradOut, xt) Î“Ă‡Ă¶ the gradient w.r.t. the weight matrix is the outer product of the output gradient and the input. This is the standard linear-layer backprop formula.

self.W -= lr * gW Î“Ă‡Ă¶ vanilla SGD with no momentum. Works for a quick prototype; would need Adam or similar for the Transformer in Phase 2.

def makeToyVideo(N=30, K=5, disagree=True):

    # Build K annotator distributions over N frames.

    # disagree=True Î“ĂĄĂ† half peak at frames 5Î“Ă‡Ă´10, half at 20Î“Ă‡Ă´25.

    scores = []

    if disagree:

        campA = np.zeros(N); campA[5:10] = 1   # "pets" annotators

        campB = np.zeros(N); campB[20:25] = 1  # "products" annotators

        for k in range(K):

            base  = campA if k % 2 == 0 else campB

            noisy = base + np.random.rand(N) * 0.3

            noisy = np.clip(noisy, 1e-6, None)

            scores.append(noisy / noisy.sum())

    return np.array(scores)





def run_config(annotator_scores, mode="mine", steps=80, T=200):

Phase 1 Results — Non-Regression Confirmed

what the numbers mean

On real TVSum data (50 videos, 20 annotators, 100 frames per video), the Wasserstein barycenter achieves held-out W² of 0.6199 vs baseline 0.6285 — a ~1.4% improvement, confirming the non-regression guarantee. The training loss for “mine” (~0.379) is numerically larger than the baseline (~0.000012) because it includes the λ·W² penalty on top of MSE — this is a units mismatch, not a failure. The held-out W² is the only metric that matters.

Phase 2 next step: replace the linear stand-in with the Transformer and benchmark on TVSum + SumMe. The geometric pipeline is validated on real data.
